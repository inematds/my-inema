Plataforma pedagógica com IA socrática

A IA nunca entrega a resposta — devolve a pergunta certa

Andaime é um classroom com três verticais funcionando ponta-a-ponta — livros ilustrados, tutor de matemática e filmes — construído sobre um princípio único: a IA espelha o esforço do aluno, nunca substitui.

O que é

Três verticais, um mesmo classroom

Professor cria a turma, o aluno entra com um código, e a IA acompanha — mas nunca resolve por ele.

📖 Andaime Junior

Ateliê de livros ilustrados pra crianças (~10 anos). Personagens, objetos, cenas e capa gerados via flux2-klein self-hosted, publicados no mural da turma e no mural global.

➗ Andaime Math

Tutor socrático de matemática. O aluno envia a tentativa, a IA devolve uma pergunta de retomada — nunca a resposta, mesmo depois de múltiplos erros.

🎬 Filmes

O storybook do Junior é renderizado server-side em MP4 1080×1920 via Remotion, com narração TTS via inemavox (engine edge).

Como funciona

Do convite ao mural

O mesmo fluxo de turma serve as três verticais — muda só o tipo de tarefa que o professor cria.

Professor cria turma→ Convida por link / código ANDA-XXXX→ Aluno entra e resolve a tarefa→ IA guia com perguntas (nunca resposta)→ Professor acompanha métricas + ★ destaca→ Publica no mural

Guardrails da IA

Pré-classificador por regex bloqueia pedidos de resposta direta; system prompt rígido; pós-validador troca respostas reveladoras por pergunta de recuo; limite de pistas configurável.

Logging pedagógico

Cada interação vira turns + event_log, dando trilha auditável de como o aluno chegou (ou não) na resposta.

Mural

/mural reúne os livros publicados globalmente; /aulas/[id]/mural é o mural público da turma, com acesso via bypass de RLS por service-role.

Pré-requisitos

O que precisa estar no ar

Stack local: Next.js + Supabase (Postgres/Auth via Docker) + Claude. Geração de imagem e voz são opcionais, self-hosted.

Node + pnpm

Node 20+ e pnpm 10+.

# confirmar versões
node -v && pnpm -v

Docker (Supabase local)

Necessário pra subir Postgres + Auth locais.

# confirmar docker no ar
docker --version

ANTHROPIC_API_KEY

Chave da Anthropic — usada pelo chat socrático, AI suggester e Math.

# adicionar no .env.local
ANTHROPIC_API_KEY=sk-ant-...

inemaimg (opcional, :8000)

Geração de imagem via flux2-klein self-hosted, usada pelo Andaime Junior (personagens, objetos, cenas, capa). Sem ele, essa vertical não gera imagens.

inemavox (opcional, :8010)

TTS + áudio self-hosted, usado na narração dos Filmes (engine edge). Sem ele, o render de MP4 fica sem narração.

Guia de uso · passo a passo

Do clone ao dashboard

Comandos reais, na ordem em que o projeto espera. Portas de Supabase local dedicadas (55321–55329) pra não colidir com outras instâncias.

1

Instalar dependências

Usa pnpm (workspace configurado em pnpm-workspace.yaml).

pnpm install
2

Configurar variáveis de ambiente

Copie o exemplo e preencha ANTHROPIC_API_KEY e as chaves do Supabase.

cp .env.local.example .env.local  # edite os valores
3

Subir o Supabase local

Sobe Postgres + Auth via Docker nas portas 55321–55329.

pnpm exec supabase start
4

Aplicar as migrações

Roda as migrações 0001–0014 (schema base, roles, chat, notificações, realtime).

pnpm exec supabase db reset
5

Gerar os tipos TypeScript do schema

Mantém src/types/database.ts sincronizado com o banco local.

pnpm exec supabase gen types typescript --local > src/types/database.ts
6

Rodar o servidor de desenvolvimento

O script dev do package.json sobe o Next.js na porta 3020, exposto em todas as interfaces.

pnpm dev  # http://localhost:3020
7

Criar conta e escolher o papel

/signup cria conta como student por padrão. Professor se promove automaticamente ao criar a primeira turma em /teacher/classes/new; pai/mãe faz self-upgrade via /api/auth/become.

# aluno entra numa turma existente com o código da turma
/student/join  # código no formato ANDA-XXXX
8

Rodar os testes e o build

Vitest para unit, typecheck e lint antes de qualquer PR; build de produção antes de deploy.

pnpm test          # unit (Vitest)
pnpm tsc --noEmit  # typecheck
pnpm lint
pnpm build
Exemplos

Dois caminhos, a mesma sala

Sem capturas de tela ainda — a estrutura real de cada fluxo, direto do código.

📖 Fluxo Andaime Junior (aluno)

  1. Abre a tarefa em /student/assignment/[id]
  2. Cria personagens, depois objetos, depois cenas (via flux2-klein)
  3. Publica o livro no mural da turma (/aulas/[id]/mural)
  4. Opcional: gera o filme do storybook (Remotion + narração TTS)

➗ Fluxo Andaime Math (aluno)

  1. Envia uma tentativa em texto livre no <MathWorkspace>
  2. POST /api/math/answer chama Claude com o problema, critério e histórico
  3. Validação dura bloqueia qualquer "a resposta é…"; a IA devolve uma pergunta de retomada
  4. Professor vê a trilha completa em /teacher/attempts/[id]
